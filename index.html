优化程序性能

发表于 2019-10-18 | 分类于 计算机系统

编写高效程序主要有两点；

  1. 选择适当的算法和数据结构；
  2. 编写出编译器能够有效优化以转换成高校可执行代码的源代码。

针对第二个优化点，就必须对编译器的编译优化有一定的掌握，同一段逻辑代码用不同方式写出来，执行起来可能会有很大差别；
为了使程序性能最大化，编译器、开发人员都需要一个目标机器模型，如何处理指令、各个操作时序性，如编译器必须知道时序信息，才能确定用一条乘法指令，还是用移位和加法的某种组合。

1. 编译器的局限

妨碍编译器优化主要有两个地方：1.内存别名使用(两个指针指向同一内存位置)； 2. 函数调用

4. 现代处理器

上面的优化不依赖于机器任何特性，只是简单降低了过程调用开销，消除了一些妨碍编译优化的因素。现代处理器引入诸如流水线、超长指令字，实现指令并行、数据并行等优点。

有两个地方限制了程序最大性能；当一系列操作必须按照严格顺序执行，会遇到1）延迟界限。(latency bound) 限制了处理器利用指令级并行的能力；2）吞吐量限制，刻画了处理器功能单元的原始计算能力，这个界限是程序性能终极限制。

4.1 整体操作

5. 一些限制因素

关键路径指明了执行该程序所需时间的一个基本下界。如果程序中某条数据相关链，这条链上所有延迟之和等于T，那么这个程序至少需要T个周期才能执行完。

5.1 寄存器溢出

循环并行性手汇编代码描述计算能力限制，如果并行度p超过可用寄存器数量，那么编译器就会溢出，将某些值放到内存中，通常是在运行时堆栈上分配空间。

滑动窗口

发表于 2019-10-03 | 分类于 算法与数据结构
leetcode76 最小覆盖子串

给你一个字符串 S、一个字符串 T，请在字符串 S 里面找出：包含 T 所有字母的最小子串。

示例：

输入: S = “ADOBECODEBANC”, T = “ABC”
输出: “BANC”

滑动窗口本质上是一种高级双指针技巧的算法框架。

  • leetcode76 最小覆盖子串(上题)
  • leetcode438 找到字符串中所有字母异位词
  • leetocde3无重复最长子串

滑动窗口算法的思路是这样：

  1. 我们在字符串 S 中使用双指针中的左右指针技巧，初始化 left = right = 0，把索引闭区间 [left, right] 称为一个「窗口」。

  2. 我们先不断地增加 right 指针扩大窗口 [left, right]，直到窗口中的字符串符合要求（包含了 T 中的所有字符）。

  3. 此时，我们停止增加 right，转而不断增加 left 指针缩小窗口 [left, right]，直到窗口中的字符串不再符合要求（不包含 T 中的所有字符了）。同时，每次增加 left，我们都要更新一轮结果。

  4. 重复第 2 和第 3 步，直到 right 到达字符串 S 的尽头。

这个思路其实也不难，第 2 步相当于在寻找一个「可行解」，然后第 3 步在优化这个「可行解」，最终找到最优解。左右指针轮流前进，窗口大小增增减减，窗口不断向右滑动。

结果图是一个以边组成的二维数组。 每一个边 的元素是一对 [u, v]，用以表示有向图中连接顶点 u and v和顶点的边，其中父节点u是子节点v的一个父节点。

返回一条能删除的边，使得剩下的图是有N个节点的有根树。若有多个答案，返回最后出现在给定二维数组的答案。

解题思路：

主要是判断有无环，再判断有没有入度为2的节点。
先来找入度为2的点，如果有的话，那么我们将当前产生入度为2的后加入的那条边标记为second，前一条边标记为first。然后我们来找环，为了方便起见，找环使用联合查找Union Find的方法，当我们找到了环之后，如果first不存在，说明是第二种情况，我们返回刚好组成环的最后加入的那条边。如果first存在，说明是第三种情况，我们返回first。如果没有环存在，说明是第一种情况，我们返回second

630 课程表三

这里有 n 门不同的在线课程，他们按从 1 到 n编号。每一门课程有一定的持续上课时间（课程时间）t 以及关闭时间第 d天。一门课要持续学习 t 天直到第 d 天时要完成，你将会从第 1 天开始。

给出 n 个在线课程用 (t, d) 对表示。你的任务是找出最多可以修几门课。

example:

输入: [[100, 200], [200, 1300], [1000, 1250], [2000, 3200]]
输出: 3
解释:
这里一共有 4 门课程, 但是你最多可以修 3 门:
首先, 修第一门课时, 它要耗费 100 天，你会在第 100 天完成, 在第 101 天准备下门课。
第二, 修第三门课时, 它会耗费 1000 天，所以你将在第 1100 天的时候完成它, 以及在第 1101 天开始准备下门课程。

第四门课现在不能修，因为你将会在第 3300 天完成它，这已经超出了关闭日期。

贪心法

先对数组以d排序，在遍历数组的过程中维护当前日期，若课程可选，那么增加当前日期，若不可选，从已经选过的课程中找出一个耗时最长的课程，若该课程耗时比当前课程长，那么替换。重复此过程。替换的过程中只需要找到最长耗时课程就可以了，因此可以用优先级队列进行优化

leetcode269 火星词典

发表于 2019-08-31 | 分类于 算法与数据结构

有新的一个单词表，字母顺序不同于我们现在使用的顺序。给你一个单词数组，请给出正确的字母顺序。

  • 假设所有字母都是小写
  • 如果顺序是不可用，就返回空
  • 如果有多个合理的顺序返回一个即可

Example 1:

Input:
[
“wrt”,
“wrf”,
“er”,
“ett”,
“rftt”
]

Output: “wertf”
Example 2:

Input:
[
“z”,
“x”
]

Output: “zx”
Example 3:

Input:
[
“z”,
“x”,
“z”
]

Output: “”

Explanation: The order is invalid, so return “”.
思路:

给一个单词字典，单词是按照字典序排序，求字母的排序。以题中例子，先看所有单词的第1个字符，可知顺序是w->e-r。然后对于两个连续的单词，找到第一个不相同的字符，比如 wrt和wrf，wr之后t在f之前，所以排序是 t->f。按照当前字母前面出现的字母个数排序，比如w前面有0个字母，e前面有w一个字母，r前面有e和w两个字母，所以排序是w->e->r。因此可以归结为一个拓扑问题，先建图，然后进行遍历。首先统计入度：w的入度是0，e的入度是1，r的入度是2。先把入度为0的节点放入结果中，然后取出w后面连接的节点，将他们的入度-1，如果有入度为0的节点，再放入结果中。

time: 建图->O(n*k), Topological sort-> O(26 + n) = O(n)，space: O(n)，主要是Map的大小，k表示单词平均长度。

在图论中，拓扑排序（Topological Sorting）是一个有向无环图（DAG, Directed Acyclic Graph）的所有顶点的线性序列。拓扑排序通常用来“排序”具有依赖关系的任务。
