1.4 调整gradle编译参数

配置gradle.properties

配置最大内存

org.gradle.jvmargs=-Xmx2048m -XX:MaxPermSize=1024m

守护进程：

org.gradle.daemon=true

并行编译：

org.gradle.parallel=true

开启缓存：

android.enableBuildCache=true

1.5 死库写法不推荐

dependencies{
compile ‘com.google.grpc:1.2+’
}

虽然可以保证每次库都是最新的，也会带来如下问题：

  • build网络检查 速度慢
  • 库更新带来bug
  • 开发者得到不同版本的包
    推荐写为固定版本

1.8 exclude关键字

剔除一些组织库

compile(‘com.google.avg’){
exclude group: ‘com.google.avg.util’ //组织名称
}

剔除某个库：

compile(‘com.android.support:appcompat-v7:23.2.0’) {
exclude group: ‘com.android.support’, module: ‘support-annotations’ // 写全称
exclude group: ‘com.android.support’, module: ‘support-compat’
}

1.9 subprojects

多模块项目之间有重复的配置，可以使用这个配置

subprojects {

version '1.0.0-SNAPSHOT'
group 'hcyxy.tech'

apply plugin: 'idea'
apply plugin: 'java'
apply plugin: 'kotlin'

sourceCompatibility = 1.8

compileKotlin {
    kotlinOptions.jvmTarget = "1.8"
}
compileTestKotlin {
    kotlinOptions.jvmTarget = "1.8"
}


repositories {
    mavenLocal()
    mavenCenter()
}

dependencies {

    testCompile "org.testng:testng:${testng_version}"

    //kotlin
    compile "org.jetbrains.kotlin:kotlin-stdlib-jdk8:$kotlin_version"

    //log4j
    compile group: 'org.apache.logging.log4j', name: 'log4j-api', version: '2.8'
    compile group: 'org.apache.logging.log4j', name: 'log4j-core', version: '2.8'

    //log
    compile 'ch.qos.logback:logback-classic:1.1.7'
    compile "org.slf4j:slf4j-api:${slf4j_version}"
    compile "org.slf4j:jul-to-slf4j:${slf4j_version}"
}
//noinspection GroovyAssignabilityCheck
tasks.withType(JavaCompile) {
    options.encoding = "UTF-8"
}}
